Bloqueando al troll..

Escrito el Sep 07, 2017 y tuvo 6 comentarios

Los que me siguen en Twitter pueden que hayan notado varios de mis Tweets relacionados con un pequeño e insignificante ataque de Trolls que SystemInside está teniendo últimamente. Este será el único y último post donde voy a tratar este tema, para comentar un poco cual es mi reacción y opinión al respecto. Además, les muestro como hacerle frente a la situación.

Me interesa menos de lo que piensan

Si querido Troll, comienzo por dejarte este mensaje. Aunque estoy redactando todo un artículo, aunque haya enviado algunos Tweets, tu opinión, tu comentario, me interesa menos de lo que piensas, de hecho, no me interesa para nada. Tal es así que automáticamente pasan a Papelera, y las IPs que has estado usando, poquito a poco van llenando mi Blacklist en el Firewall.

Por una cuestión de ética (que yo si tengo) no voy a publicar acá todas las IPs desde las que he estado recibiendo “insultos”, solo diré que además de bloquearlas, las estoy reportando, poquito a poquito, suave suavecito en sus respectivos proveedores. Quizás no sirva de mucho, pero ¿quién sabe?.

Desgraciadamente de esto se está llenando Internet. El uso del anonimato para insultar, atacar a personas o instituciones solo demuestra lo cobarde que algunos pueden llegar a ser. Yo no soy un figura pública, pero no me oculto. Mis videos están en Youtube, mi rostro queda ahí junto a mi forma de pensar, así que te invito (si tienes huevos), querido Troll, a que te llenes de valor y me des la cara por lo menos a la hora de tratar de insultarme.

Puedes poner Nicks falsos, correos electrónicos falsos, incluso la IP que estás usando quizás es un Proxy, una VPN, o sabrá Dios, pero no puedes ocultar la envidia, la molestia que estás sintiendo cuando dedicas parte de tu tiempo (al parecer no muy valioso), para dejar comentarios que sabes, nunca saldrán a la luz.

Bloqueando al troll.

Dicho esto, es posible que algunos de ustedes, que están leyendo esto, estén viviendo o hayan pasado por la misma situación. Lamentablemente no hay mucho que podamos hacer, aunque si existen algunas medidas podemos tomar. De momento, les puedo ofrecer dos formas de hacerlo, una más fácil, y otra no tan fácil pero tajante.

La forma fácil.

Si usas WordPress para tu sitio o blog, hay un método para pasar por alto determinados comentarios indeseables. En Ajustes » Comentarios hay una sección de Moderación y Lista Negra de Comentarios. Verán algo como esto:

 

En la primera sección todas las IP, Palabras, URL o Correo Electrónico que pongamos, se mantendrán en estado de “Pendiente” para ser moderado. Debemos poner cada IP, Palabra, etc.. separadas una a una por línea.

En la segunda sección es lo mismo, solo que lo que se ponga ahí pasará directamente a Papelera, con lo cual en teoría, ni lo veremos a no ser que vayamos a ese lugar.

La forma no tan fácil

El segundo método, que también estoy usando es un poco más complicado, y solo podremos llevarlo a cabo si nuestro proveedor de servicios nos permite usar reglas de IPTables. IPTables en sí, es un poco complicado de usar, pero hay herramientas que nos facilitan el trabajo.

En mi caso para no complicarme la existencia hago uso de Firehol. Esta herramienta se encuentra en los repositorios de la mayoría de las distribuciones y es mucho más amigable de configurar. En el caso de Ubuntu Server:

# apt install firehol

Una vez instalado lo que haremos es editar el fichero /etc/firehol/firehol.conf y le pondremos adentro lo siguiente:

# FireHOL configuration file
#
# See firehol.conf(5) manual page and FireHOL Manual for details.
#
# This configuration file will allow all requests originating from the
# local machine to be send through all network interfaces.
#
# No requests are allowed to come from the network. The host will be
# completely stealthed! It will not respond to anything, and it will
# not be pingable, although it will be able to originate anything
# (even pings to other hosts).
#

version 6

# Aceptamos todo el tráfico desde cualquier interfaz de red
interface any world

# Por defecto a todo el mundo lo rechazamos
policy reject

# Bloqueamos determinadas IPs o Subredes por el puerto 80
server4 http reject src X.X.X.X
server4 http reject src X.X.0.0/24

# Bloqueamos determinadas IPs o Subredes por el puerto 443
server4 https reject src X.X.X.X
server4 https reject src X.X.0.0/24

# Para el resto, aceptamos que accedan
server http accept
server https accept

client all accept

 

Y esta sería una configuración bastante básica para efectiva. Tendríamos que bloquear o abrir determinados servicios o puertos según los usemos. Si estamos usando un VPS, también tenemos aceptar poder acceder vía SSH, ya sea a nuestra IP o a todas en general.

Una vez que editemos el fichero, echamos a andar el Firewall con el comando:

# firehol start

Y para que se inicie siempre que reiniciemos el servidor, editamos el fichero /etc/default/firehol y ponemos en YES la línea:

START_FIREHOL=YES

Lo malo de esto

Lo malo, por no decir lo jodido de esto, es que aunque podemos bloquear todo un rango de IP, es posible que alguien que no esté molestando se vea afectado, con lo cual, tendremos que ir añadiendo las IPs manualmente.. pero oye, eso es parte de la diversión ¿o no?
