Além disso, com essa configuração, desconfigurou o teclado para o gnome. O teclado ficou parecido com a configuração de um teclado americano, mesmo nas configurações do gnome estar configurado para abnt2.

não sei se ajuda, mas dentro do gnome, (em qualquer configuração do rc.conf), se eu executar o seguinte comando:

setxkbmap -model abnt2 -layout br

o meu teclado fica configurado corretamente (não que ele já não esteja correto no gnome por default)

Re: [RESOLVIDO] [teclado] Problemas com a configuração para tty

coloque no .bashrc
setxkbmap -model abnt2 -layout br

ou

vai em /etc/X11/xorg.conf.d

edite o arquivo 10-evdev.conf

adicione   Option   "XkbLayout" "br" em
Section "InputClass"
        Identifier "evdev keyboard catchall"
        MatchIsKeyboard "on"
        MatchDevicePath "/dev/input/event*"
        Driver "evdev"
EndSection

parecido com:

Section "InputClass"
        Identifier "evdev keyboard catchall"
        MatchIsKeyboard "on"
        MatchDevicePath "/dev/input/event*"
        Driver "evdev"
        Option   "XkbLayout" "br"
EndSection

mas pelo sim, pelo não, awe vai os procedimentos necessários para resolver o problema:

copiar a configuração de teclado br-abnt2 para uma pasta qualquer:

cp /usr/share/kbd/keymaps/i386/qwerty/br-abnt2.map.gz ~/

descompactar :

gzip -d ~/br-abnt2.map.gz

acrescentar o seguinte comando no final da linha:

altgr keycode 16 = slash
altgr keycode 17 = question

recompactar e mover o arquivo de volta para local inicial:

gzip ~/br-abnt2
mv br-abnt2.map.gz /usr/share/kbd/keymaps/i386/qwerty/br-abnt2-note.map.gz

sem contar q tem q configurar o rc.conf...
